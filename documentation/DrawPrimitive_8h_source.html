00050         // Absolute X,Y position of the alias.  Not used for "base" objects, though
00051         // it is harmless to set it for them.
00052         //
00053         // There are several reasons why Alias contains ABSOLUTE coordinates instead
00054         // of RELATIVE coordinates, all of which relate to GroupPrimitive objects:
00055         //
00056         // 1. Groups don't know where they are.
00057         //
00058         // GroupPrimitive objects neither contain nor use coordinate information;
00059         // they merely contain other DrawPrimitive objects that contain absolute
00060         // coordinates and therefore know where they are.  If aliases within a group
00061         // contained relative coordinates, they would need to know the position to
00062         // which they were relative, which means that groups would have to know their
00063         // positions (or be able to get it from their Parents--see below).
00064         //
00065         // 2. The need for "parent" information.
00066         //
00067         // Giving groups positional information is useless unless DrawPrimitives
00068         // know the groups to which they belong.  That means adding something like
00069         // a Parent field, and all the necessary code to build this information and
00070         // maintain its integrity.  It would also render certain information
00071         // meaningless, such as LinePrimitive (X1,Y1) coordinates, hyperlink
00072         // coordinates, and starting coordinates for curves, polycurves, etc.
00073         //
00074         // 3. SPEED!
00075         //
00076         // Many calculations have to be performed in real-time in which it is
00077         // necessary to know where an alias object's coordinate actually is.  For
00078         // instance, dragging a polycurve node requires that the program know where
00079         // that node is in real-time.  When alias objects contain absolute
00080         // coordinates, they can simply refer to their base objects and do a fast
00081         // calculation to find out the actual position.  When they contain relative
00082         // coordinates, however, they first have to walk a parent tree.  I felt that
00083         // this would take far too much time for many operations.
